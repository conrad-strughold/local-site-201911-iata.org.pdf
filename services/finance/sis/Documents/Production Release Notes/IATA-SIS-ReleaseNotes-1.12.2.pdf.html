8863558

 

Minor

 

Problems with 

incoming invoice

 

Area: IS WEB 

 

Issue

: 

As per 
‘IATA_IS_XML_Invoice_Standard_V3.9.xsd ' 
there exist a validation on field 
‘BankAccountNo’ which restricts values to 
only characters and numbers. 
Since  the  field  ‘Bank  Account  number’  in 
the file in question contains hyphen [-] and 
dot [.] it failed in XSD validation.

 

 

Corrective Action: 
The 

XSD 

validation 

on 

field 

‘BankAccountNo’  is  modified  to  accept 
ASCII characters between 32 to 126. 
Since there was change in XSD file so as per 
discussion  with  IATA  version  number  of 
XSD’s is updated from V3.9 to V3.9.0.1 
 

 
 
 
 
 
 
 
 
 
 
 
 

1.12.2 

 
 
 
 
 
 
 
 
 
 
 
 

8946484   

Minor 

Wrong headers in 

the csv report 

Area: IS WEB 
 
Issue: 
The  issue  was  identified  in  Reference  Data 
file  generated  for  complete  contact  data. 
The  column  headers  text  for  columns 
‘Member  Legal  Name’  and  ‘Member 
Commercial  Name’  were  printed  as  ‘M3’ 
and ‘M4’ respectively. After investigating we 
found  that  the  issue  was  because  of 
incorrect mapping of column header names 
in code. 

The issue is only related to column headers, 
the  data  (i.e.  Member  Legal  Name  and 
Member  Commercial  Name)  is  correctly 
printed in these columns. 

Corrective Action: 
The  code  changes  have  been  done  to  fix 
this  issue.  The  column  headers  are  now 
correctly  mapped.  With  this  fix,  the  report 
will  now  print  column  headers  as  ‘Member 
Legal  Name’  and  ‘Member  Commercial 
Name’.

Minor

 

File pending in 

Production

 

Area: IS WEB 

 

Issue: 

There was an issue in the validation process 
of MISC IS-XML. 
The  file  failed  in  validation.  This  issue  is 
specific 

to 

Correspondence 

Invoice 

submitted using IS-XML. On investigation, it 
was  found  that  the  member  provided  an 
incorrect 

correspondence 

reference 

number for the Correspondence invoice.  
There  is a bug found in Validation process, 
where if correspondence reference number 
is  wrongly  provided,  the  process  was  still 
performing amount  related validation. This 
case was not handled in the application and 
was getting failed.

 

Corrective Action: 
 
The  code  changes  have  been  done  to  fix 
this  issue.  With  this  fix,  when  incorrect 
correspondence  reference  number  is 
provided, 

system 

will 

mark 

the 

correspondence 

invoice 

to 

error-

correctable  invoice  and  no  further  amount 
validations  will  be  performed  by  the 
system.  
System  will  provide  a  new  error  message 
for  correctable  correspondence  reference 
number  field  'Correspondence  Reference 
Number  provided  in  the  Correspondence 
Details  section  does  not  exist  in  the  audit 
trail.' 
Upon  correction  of  the  invoice  from  error-
correction 

screen, 

further 

amount 

validations will be performed
