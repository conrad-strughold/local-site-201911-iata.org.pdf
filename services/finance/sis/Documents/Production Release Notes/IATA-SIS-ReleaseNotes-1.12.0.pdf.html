Area: ISWEB 

 

Issue: 
ISIDEC/ISXML file process was taking more 
than  expected  time.  The  issue  has  started 
occurring  after  Release  1.12  deployment. 
On  investigation  it  was  found  that  the  files 
were  taking  more  time  during  validation 
process.  The  issue  has  occurred  due  to  a 
code change for NII security fix.  
Under  this  fix  we  converted  String  variable 
to SecureString wherever we store sensitive 
information  like  database  password  etc. 
After  this  fix,  system  parameter  values  also 
started passing through SecureString. Since 
the  system  parameter  value  contains  large 
set  of  bytes  and  was  getting  fetched 
multiple  times  by  validation  process  to 
validate  the  single  file,  the  process  started 
taking more time.  

The  issue  was  that  Validation  Process  was 
fetching  system  parameter  value  multiple 
times while processing a single file.   
 

Corrective Action: 
 
Validation  Process code has been changed 
to  fetch  system  parameter  only  once  for  a 
file
