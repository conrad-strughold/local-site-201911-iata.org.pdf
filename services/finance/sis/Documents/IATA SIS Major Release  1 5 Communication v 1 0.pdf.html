Sync Master and 
Member Profile data 
from Production to

Description:  

Currently, there is no automated process that synchronizes Member Profile, Blocking Rules and selected Master data from

Selected fields from the Member Profile, selected Master Tables and ICH/ACH Blocking Rules data will be synced on a daily

573 

User to be able to 
reply to a 
Correspondence by 
clicking the URL in 
the Correspondence 
Alert email 

Description:  

When a user clicks the URL (hyperlink) in a correspondence email alert, he/she views the correspondence screen, but the 
‘Reply’ button is not shown. This forces the user to access the correspondence from the Billing History and Correspondence 
screen in order to reply to the correspondence.  

This CMP will address this requirement. Clicking the URL in the correspondence email alert will show the correspondence 
screen along with the ‘Reply’ button, provided the correspondence is in a state where the user can reply, and the user has the 
required permissions. 

 
Applicability: 

1.  Passenger / Cargo / Miscellaneous view/initiate correspondence screen 

Changes: 

1.  No change required for SIS Members

2.  When the view/initiate correspondence screen is accessed by a user by clicking the URL in the correspondence email 

alert, the ‘Reply’ button will be visible.  The user will be able to reply to the correspondence, provided the following pre-
conditions are true: 

 

a.  The instance/stage of correspondence for which the URL has been clicked is the last in the entire correspondence’s 

chain; i.e. NO further stage exists, even in a ‘Saved’ or ‘Ready for Submit’ Sub-status 
 

b.  The Status of the correspondence is ‘Open’ 

 

c.  The Sub-Status of the correspondence is ‘Received”.  

 

d.  The correspondence cannot be expired (i.e. the Expiry Date of the correspondence cannot be equal to or later than 

the system time in UTC) 
 

e.  The user who has clicked on the URL of the email alert (and accessing the screen) has the following Permissions: 

 

i. Passenger.BillingHistory.ViewCorrespondence, 

and 

 

ii. Passenger.BillingHistory.DraftCorrespondence 

 

589 

Scaling Factor to be 
included in invoice 
PDF 

Description:  

The Miscellaneous PDF invoice does not display the Scaling Factor. As a result of this, when the Scaling Factor is greater than 
one, the viewer of the PDF cannot understand the basis on which the ‘Base Amount’ (charge amount) was calculated.  

As part of this CMP, the Scaling Factor will be displayed when it is greater than one. 

 
Applicability: 

1.  All Miscellaneous billing transactions 

 

Changes: 

1.  Miscellaneous Invoice PDF 
 

a.  Scaling Factor will be shown below the Unit Price field, if the Scaling Factor is greater than 1  

 

b.  Example 1: If Unit Price is 263.125 and the Scaling Factor is 1, then value of field Unit Price will be shown as 263.125 

 

c.  Example 2: If Unit Price is 263.125 and the Scaling Factor is 2, then the value of field Unit Price will be shown as 

263.125 @ SF 2. In such cases, information about Scaling Factor will be shown below the Unit Price; i.e. Unit Price 
and Scaling Factor will be shown in 2 different lines within the Line Item row

597 

SIS to generate 
weekly reference 
data and contacts 
CSV 

Description:  

To ensure that Billing Members stay updated with the latest applicable Reference Data and Contacts of other Members, a new 
CSV file will be available containing such information. 

Members wishing to receive such files have any or all of the following options: 

1.  Receive information only about changes in Reference Data, that are applicable at the start of a Billing Period  

2.  Receive complete Reference Data of all Members, that is applicable at the start of a Billing Period (irrespective of any 

changes were applicable or not)   

3.  Receive complete Contacts Data of all Members, that is applicable at the start of a Billing Period (irrespective of any 

changes were applicable or not) 

Please note that a Member making changes to their own Reference Data will receive information about additions/changes 
made to its Locations in the update file(s). Similarly, a Member will receive information about its own Contacts.  This report will 
be available at the start of each period.

Structure and specific requirements of file type “Complete Contacts Data” 

The structure and contents of this file will be identical to the Contact Details CSV file offered for download by the system in the IS-WEB Member and Contact 
Report, when: 

1.  No filter criteria have been defined anywhere, i.e.: 

a.  Member Name is not defined 

b.  Country is not defined 

c.  ACH Members or ICH Members, Dual Members, Non-CH Members, IATA Members are unchecked 

d.  Contact Name is not defined 

e.  Email is not defined  

2.  Tabular Format is chosen as the Report Display Option 

3.  All ‘Available Fields’ have been selected/moved into ‘Selected Fields’ 

4.  No sort criteria have been defined
