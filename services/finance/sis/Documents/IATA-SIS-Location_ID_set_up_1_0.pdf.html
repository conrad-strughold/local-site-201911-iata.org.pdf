1.  Objective 

 

The purpose of this document is to:  

1.

 

Provides information to members on how to set up Location ID(s)  
 

2.

 

Assigning Location ID to user: Provides information on the various setting changes that has be 
looked on to assign a Location ID to user. The aim is to have segmentation in information so that 
the user can see only the invoices from a particular Location ID.  
Also provides information on setting up alerts so that user is only notified of invoices that is 
associated to a specific ID.  

3.

 

Flow process: The various output options of the system is described using a process flow chart. And 
also the settings that is required to attain these outputs are described.  

2.  Setting up location ID(s)  

 

When SIS Ops creates a profile for new member their Location ID 

– ‘Main’ is already filled 

in with the information such as the Member Commercial Name, Company registration ID, 
complete address of the head office, bank details (Which is provided to IATA via an 
authorized form after signing ISUA) etc. from the IATA data base. 

This information can be viewed or edited from:  

Profile and User Management -> Manage Member Profile  

 

Then Locations -> Search Existing Location -> Main 

 

Click on ‘View’ to see the complete details.

4.  Process Flow 

 

 

 

 

 

 

Outputs are in the form of ISXML and Offline Archives (OARs). 

 

Outputs can be: 

1.

 

Weekly basis (MXMLT-A89YYYYMMPP.zip and OAR-MISC-P-A89-YYYYMMPP.zip) 

Billing month and period. Represented as YYYYMMPP. E.g. 20160104 = Period 4 of 
January - 2016. 

2.

 

Daily basis (D-MXMLT-A89YYMMDD.zip and D-OAR-MISC-P-A89-YYYYMMDD.zip) 

YYYYMMDD will be equal to ‘Target Date of Daily Delivery Output’ set during the daily 
scheduled output process. 

3.

 

Weekly basis per location ID (i.e. 30) 

(MXMLT-A89YYYYMMPP-L30.zip and OAR-MISC-P-A89-YYYYMMPP-L30.zip) 

-L30 indicate a specific location ID 

4.

 

Daily basis per location ID (i.e. 30) 

(D-MXMLT-A89YYMMDD-L30.zip and D-OAR-MISC-P-A89-YYYYMMDD-L30.zip) 

-L30 indicate a specific location ID 

 

The Offline Archives Output has options to include: 

1.

 

Invoice PDF 

2.

 

Details Listing – This output displays further details of the invoice for billing and billed entity 

3.

 

Supporting Documents – Documents that were attached to the invoice when it was uploaded    

to the system.  

4.

 

Digital Signature files

6. 

Bilateral function is activated

 

 

Every day an output file is produced and send to the airline (Only when there is a minimum 
of one invoice submitted by a supplier for that airline). 

The output file includes all bilateral invoices including E&F invoices paid through BSP (SMI 
‘S’). 

The output file is send to the default Misc iiNET setup account.  

 

  

 

 

 

 

 

Maximum 7 files per week (when a minimum of one invoice is received per day). 

No minimum file.
