7.

IATA Publication demonstrated Cargo-XML scorecard proposal and group 
provided following feedback:

a.

Should be extended to all stakeholders including IT companies.

b.

One IT company should be able handle/manage multiple airlines (update data for)

c.

Toolkit can include a link to the scorecard

d.

Each record must have a date when it was updated

e.

Upon update, a preview must be shown before data is published

f.

Vetting/Registration process need to be in place

g.

Customer type must be included

h.

View should be instantly available for public while update is private (password protected)

i.

Notification service will be included as well as there will be a link to view the recent updates

j.

GDPR issue must be addressed

k.

Data Export must be possible from the tool

l.

Pricing discussion is out of scope

m.

Latest version will be displayed on top however option will be there to drill down

n.

EPIC and scorecard can be merged

Summary of Discussions (5/11)

13.

Flight Plan/Load Advise Message was discussed.

a.

Group noted a big difference between load advise for PAX flight and a Freighter. 

b.

Freighter is more complicated due to special load, live animals, pharma etc. 

c.

It was noted that airline send FBL and then loading instructions are provided in free text email

d.

Current examples are straight forward and simple. These doesn’t contain information 

exchanged through email such as available capacity etc.

e.

Group reviewed some complex examples shared by ALHA Airport.

f.

Next step to check with ICHC if:

i.

an updated XFBL messages with additional information or

ii.

dig down to details and try to capture complete requirements.

14.

NOTOC

a.

Airlines are using customized NOTOC messages

b.

Standardizing the NOTOC is challenging but achievable. 

c.

Not only DG but special cargo, live animal, crew container, val cargo etc.

d.

A detailed analysis of existing customized NOTOCs will be required.
